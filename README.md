# Information_Security_Incident_Management_System
Внутренняя корпоративная система для: регистрации инцидентов ИБ, классификации и приоритезации, контроля сроков, отчётности и аудита
## Идея системы
Система управления инцидентами информационной безопасности.  
Данная система необходима для регистрации, обработки и контроля инцидентов информационной информации, для обеспечения реагирования и для соблюдения установленных сроков устранения инцидентов.  
Система включает:
- Регистрацию инцидентов ИБ
- Классификацию инцидентов
- Назначение ответственных
- Изменение и контроль статуса
- Хранение истории
- Формирование отчетов
  
Пользователи системы:
- Аналитик ИБ. Регистрация, анализ инцидентов
- Инженер ИБ. Получение задач и фиксация статуса
- Руководитель ИБ. Контроль сроков и нагрузки
- Аудитор. Доступ к отчетам
- Внешняя система монниторинга. Передача данных об инцидентах

Инцидент информационной безопасности регистрируется в системе аналитиком ИБ или автоматически из внешней системы мониторинга. Инцидент классифицируется, ему присваивается приоритет и назначается ответственный инженер. В ходе обработки фиксируются действия, комментарии и изменения статусов. Руководитель ИБ контролирует процесс и формирует отчётность.

## Требования к системе
Требование к системе делится на два вида: функциональные и нефункциональные  
Функциональные: 
- Регистрация инцидента
- Классификация инцидента
- Назначение ответственного
- Управление статусами инцидента
- Ведение истории инцидента
- Контроль сроков 
- Просмотр и поиск инцидентов
- Формирование отчётности
  
Нефункциональные:
- Производительность
- Надёжность
- Безопасность
- Аудит
- Доступность
- Масштабируемость
